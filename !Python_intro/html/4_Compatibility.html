SQL¶

You can query data in Python using SQL. Many versions of SQL are compatible with Python, but I recommend SQLite because it is serverless and self-contained. This means you do not need to install/run a special SQLite server to use a SQLite database.

In [2]:
import pandas as pd
import sqlite3

con = sqlite3.connect("database.db")
In [3]:
df = pd.read_sql_query("SELECT * FROM purchases", con)

MATLAB¶

It is possible to read and write MATLAB files using the functions scipy.io.loadmat and scipy.io.savemat.

R¶

It seems like, if there is a need to, you can do just about anything in R within the Python/Jupyter environment.

SAS¶

There is a SAS kernel in Jupyter as well. At the very least, it is possible to use Jupyter to store the input and output from SAS sessions.

Excel¶

Excel integration also exists. Python can read, write, and modify workbooks using a library named openpyxl. There are Excel add-ins (xlwings, pyxll) that allow the use of Python within Excel. All seem increasingly popular.

Local Files¶

We can use Python to add a border to the local image shown below.

In [6]:
# import image packages
from PIL import Image, ImageOps

# function to add border to image
def add_border(input_image, output_image, border):
    img = Image.open(input_image)
    
    if isinstance(border, int) or isinstance(border, tuple):
        bimg = ImageOps.expand(img, border=border)
    else:
        raise RuntimeError('Border is not an integer or tuple!')
    
    bimg.save(output_image)

# call function to create & save image
if __name__ == '__main__':
    in_img = 'C:/Users/natha/!Python_intro/SP500.jpg'
    add_border(in_img, 
               output_image='C:/Users/natha/!Python_intro/SP500_border.jpg',
               border=(10,10))
